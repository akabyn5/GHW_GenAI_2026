# Build with GitHub Copilot — MLH Global Hack Week GenAI '25

## Challenge
GitHub Copilot Quickstart — Build a Simple Application

## Description
This submission showcases an application built entirely with the assistance of **GitHub Copilot** as part of MLH Global Hack Week GenAI 2025. The challenge tasked us with using Copilot prompts to generate a simple but functional project — demonstrating how an AI coding assistant can take a plain-language description and scaffold a working application from scratch. Our team used Copilot to build a **template portfolio website** with Next.js and React, guiding the AI through component structure, styling, and content layout using targeted prompts throughout the process.

## Features
- **Hero section** generated by Copilot with a name, role title, and call-to-action button
- **Projects grid** scaffolded via prompt, displaying project cards with title, description, and GitHub link
- **Skills section** built by describing the desired layout in plain English and letting Copilot produce the component
- **Contact form** with name, email, and message fields, fully generated through Copilot Chat
- **Responsive design** applied across all sections using Tailwind CSS, suggested and written by Copilot
- Entire codebase **reviewed, refined, and iteratively improved** through follow-up Copilot prompts

Open [http://localhost:3000](http://localhost:3000) in your browser to see the portfolio running.

### How We Used Copilot to Build This
1. Opened the project in **GitHub Codespaces** with Copilot enabled
2. Started with a high-level prompt in **Copilot Chat**:
   > *"Create a portfolio website using Next.js and Tailwind CSS with a hero section, projects grid, skills list, and contact form."*
3. Accepted, refined, and iterated on suggestions using **inline completions** and follow-up chat prompts
4. Asked Copilot to explain each generated block before moving on, to understand what was being added
5. Used Copilot to write **responsive CSS utilities** and fix layout issues described in plain language

## What We Learned
Building a full application through prompts made us think differently about how we communicate intent to a machine. Writing a good Copilot prompt is closer to writing a clear technical requirement than typing a search query — the more specific and contextual the prompt, the more accurate and production-ready the output. We also learned that Copilot works best as a collaborator rather than a replacement: it handles the boilerplate and structural scaffolding rapidly, freeing us to focus on the product decisions that actually require human judgment.
